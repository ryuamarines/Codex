# Share Plan

Live Log の共有は、個別記録や画像をそのまま外へ出すのではなく、まず集計結果を安全に共有する方針で進めます。

## 目的

- 個人用アーカイブの非公開性を崩さない
- SNS やメッセージアプリへ貼りやすい出力だけ先に整える
- 画像原本や個別メモを共有対象に混ぜない

## Phase 1: 画像共有の整備

対象:

- 集計タイルの PNG 共有
- 年別まとめの PNG 共有

方針:

- 共有対象は「今画面に見えている集計結果」のみ
- 個別公演のメモや画像は含めない
- スマホでは共有シート、PC では PNG 保存を基本にする

## Phase 2: 共有スナップショット

対象:

- 年別まとめ
- 参加本数
- アーティスト別推移
- 会場 / 地域のランキング

方針:

- 記録データそのものではなく、共有専用に整形した集計だけを出す
- 共有物は読み取り専用
- 共有対象に個別画像は含めない

## Phase 3: 共有 URL

条件:

- Phase 1, 2 が安定してから
- 共有対象と非共有対象の境界が UI 上でも明確になってから

方針:

- 共有 URL は「共有用スナップショット」に限定する
- 個別記録ページや Drive 画像への導線は共有しない
- 認証不要で見せる場合も、公開データは集計に限定する

## 今はやらないこと

- 個別のライブ記録ページの公開
- Google Drive 上の画像原本の共有
- 写真付きの公開タイムライン化
- SNS 的なプロフィール公開

## 実装優先度

1. 既存の PNG 共有を使いやすくする
2. 年別まとめ / 集計の共有出力を安定させる
3. 共有スナップショット用の専用 UI を考える
4. 必要なら共有 URL を設計する
