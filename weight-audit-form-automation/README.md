# 減量監査フォーム自動化 MVP

スマホから毎日 1 回だけ入力し、Google 側にデータを蓄積し、週 1 回だけ振り返るための最小構成です。

構成要素は以下の 4 つです。

1. Google フォーム
2. Google スプレッドシート
3. Google Apps Script
4. 時間主導トリガーによる週次メール送信

## 1. Google フォームの質問一覧

フォーム名: `減量監査ログ`

説明文の例:

```text
毎日 1 回だけ入力してください。30 秒で終わる想定です。
予定外回数 = 朝昼夜の予定された食事以外に食べた回数です。
```

質問は以下の順番にしてください。

| No | 質問タイトル | 形式 | 必須 | 選択肢 / 入力例 |
|---|---|---|---|---|
| 1 | 記録対象日 | 日付 | 必須 | 2026-04-13 |
| 2 | 元々の予定 | プルダウン | 必須 | 在宅 / 外出 / 飲み会 |
| 3 | 体重 | 記述式（数値） | 任意 | 87.4 |
| 4 | 歩数 | 記述式（数値） | 必須 | 6231 |
| 5 | 摂取カロリー | 記述式（数値） | 任意 | 2200 |
| 6 | 飲み会 | プルダウン | 必須 | 0 / 1 |
| 7 | 酒 | プルダウン | 必須 | 0 / 1 |
| 8 | 予定外回数 | プルダウン | 必須 | 0 / 1 / 2 / 3以上 |
| 9 | メモ | 記述式（短文） | 任意 | たん清、つい買い、在庫、打ち上げ |

### フォーム作成時の設定

スマホ入力を軽くするため、以下を推奨します。

1. 各質問は 1 画面で見やすい短い文言にする
2. `記録対象日` はその日を選ぶ。深夜に前日分を入れるときだけ前日を選ぶ
3. `元々の予定` `飲み会` `酒` `予定外回数` はプルダウンにする
4. `体重` は `測れない日は空欄可` としておく
5. `歩数` は説明欄に `整数で入力` と書く
6. `摂取カロリー` は `分かる日だけ入力。飲み会は空欄可` と書く
7. 回答後に `別の回答を送信` を押しやすい構成にする
8. Android ではフォーム URL を Chrome で開いて `ホーム画面に追加` する

## 2. スプレッドシート列設計

フォームをスプレッドシートに連携したら、回答先シート名を `log` に変更してください。

最低限の列は以下です。

| 列 | 項目名 | 内容 |
|---|---|---|
| A | timestamp | Google フォーム自動タイムスタンプ |
| B | 記録対象日 | フォーム入力の日付 |
| C | date | Apps Script で使う正規化日付 |
| D | 元々の予定 | 在宅 / 外出 / 飲み会 |
| E | 体重 | 数値 |
| F | 歩数 | 数値 |
| G | 摂取カロリー | 任意の数値。空欄可 |
| H | 飲み会 | 0 / 1 |
| I | 酒 | 0 / 1 |
| J | 予定外回数 | 0 / 1 / 2 / 3以上 |
| K | メモ | 任意 |
| L | 週番号 | Apps Script 補助列 |
| M | 予定外回数_数値 | `3以上` は `3` として扱う |
| N | 飲酒日フラグ | 0 / 1 |
| O | 日タイプ集計キー | 元々の予定をそのまま保持 |

### 補足

Google フォーム連携直後は、実際の列順が `timestamp, 記録対象日, 元々の予定, 体重...` のようになることがあります。  
今回の `weekly_summary.gs` はヘッダー名で列を判定するため、まずはそのままでも動きます。  
不足している `date` と補助列は、`initializeLogSheet()` 実行時に右端へ自動追加されます。

## 3. Apps Script 完全コード

Apps Script エディタに、以下のファイルをそのまま貼り付けてください。

- [weekly_summary.gs](/Users/Ryu/Documents/Codex/weight-audit-form-automation/weekly_summary.gs)
- [appsscript.json](/Users/Ryu/Documents/Codex/weight-audit-form-automation/appsscript.json)

コードの役割は以下です。

1. `initializeLogSheet()` で `date` と補助列を整える
2. `runWeeklySummaryManually()` で手動テスト送信する
3. `createWeeklySummaryTrigger()` で週次トリガーを作る
4. `sendWeeklySummary()` が週次サマリー本文を作って Gmail 送信する

## 4. 初期導入手順

1. Google フォーム `減量監査ログ` を作成する
2. 上の質問一覧どおりに質問を追加する
3. `回答` タブからスプレッドシートを新規作成する
4. 作成された回答シートのタブ名を `log` に変更する
5. フォームの質問タイトルが以下と一致していることを確認する

```text
timestamp | 記録対象日 | date | 元々の予定 | 体重 | 歩数 | 摂取カロリー | 飲み会 | 酒 | 予定外回数 | メモ | 週番号 | 予定外回数_数値 | 飲酒日フラグ | 日タイプ集計キー
```

`date` 以降の補助列は空でも問題ありません。`initializeLogSheet()` 実行時に自動で補われます。  
`体重` と `摂取カロリー` は空欄でも集計が止まらないようにしてあります。

6. スプレッドシートで `拡張機能 > Apps Script` を開く
7. デフォルトのコードを削除する
8. `weekly_summary.gs` の内容を貼り付ける
9. `appsscript.json` をマニフェスト表示で置き換える
10. `CONFIG.RECIPIENT_EMAIL` を自分のメールアドレスに変更する
11. 必要に応じて `CONFIG.TRIGGER_WEEKDAY` と `CONFIG.TRIGGER_HOUR` を変更する
12. `initializeLogSheet()` を一度実行する
13. 権限承認を完了する
14. `runWeeklySummaryManually()` を実行してテストメールを確認する
15. 問題なければ `createWeeklySummaryTrigger()` を実行する

## 5. トリガー設定手順

### 方法 A: コードから自動作成

1. Apps Script で `createWeeklySummaryTrigger()` を実行する
2. 既存の同名トリガーがあれば自動で整理される
3. 毎週 1 回、指定曜日・指定時刻に `sendWeeklySummary()` が動く

### 方法 B: 画面から手動作成

1. Apps Script 左側の `トリガー` を開く
2. `トリガーを追加` を押す
3. 実行関数に `sendWeeklySummary` を選ぶ
4. イベントのソースに `時間主導型` を選ぶ
5. 週ベースのタイマーを選ぶ
6. 曜日と時間帯を指定する

### 推奨

- 月曜朝運用: `MONDAY` + `7`
- 日曜夜運用: `SUNDAY` + `21`

月曜朝に送るなら、集計終了日を前日の日曜にするため `SUMMARY_END_OFFSET_DAYS = 1` のままで使えます。  
日曜夜に送る場合は `SUMMARY_END_OFFSET_DAYS = 0` に変更してください。

## 6. 想定メール文面テンプレート

件名:

```text
減量監査 週次サマリー 2026-04-12
```

本文例:

```text
対象期間: 2026-04-06 ～ 2026-04-12
平均体重: 86.9kg
前週差: -0.4kg
平均歩数: 6240歩
平均摂取カロリー: 2210kcal（入力 5日）
飲み会日数: 2日
飲酒日数: 3日
予定外回数 合計: 5回
予定外回数 2以上の日数: 1日
崩れやすい日タイプ: 在宅
コメント: 在宅日に予定外摂取が集中。飲酒日は許容範囲。
```

## 8. 運用メモ

MVP では以下だけ守れば十分です。

1. 毎日 1 回だけ入れる
2. 予定外回数を最重要 KPI として見る
3. 酒と飲み会は分けて記録する
4. コメントは気になる日だけでよい
5. 週次メールで全体傾向だけ確認する

この構成なら、ChatGPT 内にログをためず、Google 側だけで完結します。
